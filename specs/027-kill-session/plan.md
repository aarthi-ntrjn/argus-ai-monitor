# Implementation Plan: Kill Session Button

**Branch**: `027-kill-session` | **Date**: 2026-04-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/027-kill-session/spec.md`

## Summary

Add a kill button to each session card (next to the "View Details" link) and to the session detail page header. Clicking it opens a confirmation dialog. On confirmation, the frontend calls the existing `POST /api/v1/sessions/:id/stop` endpoint to terminate the process via OS APIs. The button shows loading, success, and error states. This is a frontend-only feature: the backend stop API, process termination logic, and frontend API wrapper (`stopSession()`) already exist.
