# Implementation Plan: Fix JSONL Parsing Differences Between Claude and Copilot

**Branch**: `037-fix-jsonl-parsing` | **Date**: 2026-04-18 | **Spec**: [spec.md](spec.md)

## Summary

Fix six behavioral differences between the Claude Code and Copilot JSONL watchers: stabilize output entry IDs so INSERT OR IGNORE deduplication actually works; apply the same tail-read strategy to both watchers on every attach; remove the destructive output-clear from the Copilot watcher; resume sequence counters from the stored maximum on re-attach; convert Copilot's sync file I/O to async; and extract the shared tail-window constant. No schema changes are needed.

## Technical Context

**Language/Version**: TypeScript 5.x / Node.js 20+
**Primary Dependencies**: better-sqlite3, chokidar, fs/promises, Jest
**Storage**: SQLite — existing `session_output` schema unchanged; new `getMaxSequenceNumber` query added
**Testing**: Jest (existing suite in `backend/tests/unit/`)

**Performance Goals**: ≥10 concurrent sessions; file watch latency < 1 s per change event
**Constraints**: No breaking changes to HTTP API or DB schema; all existing tests must continue to pass
**Scale/Scope**: Single user, ≤10 active sessions

## Phase 0: Research

All unknowns resolved from code inspection. See [research.md](research.md) for full findings.

**Key decisions made**:
1. Claude IDs: `${entry.uuid}-u-N` / `${entry.uuid}-a-N` (role + block index suffix)
2. Copilot IDs: `${sessionId}-${lineByteOffset}` (absolute file byte offset)
3. Both watchers: tail read (`max(0, fileSize - TAIL_BYTES)`) on every attach
4. Sequence counter: initialized from `getMaxSequenceNumber(sessionId) + 1` on every attach
5. `TAIL_BYTES = 16 * 1024` extracted to `backend/src/utils/watcher-constants.ts`
6. Remove `deleteSessionOutput` from `watchEventsFile`; no replacement
7. Copilot `readNewLines` converted to async (fs/promises)

## Phase 1: Design

### Implementation Approach

Six logical change sets, ordered by dependency:

#### Change Set A — Shared Constant (no dependencies)

Create `backend/src/utils/watcher-constants.ts`:

```typescript
/** Maximum bytes read from a JSONL file on initial watcher attach. */
export const TAIL_BYTES = 16 * 1024;
```

No other changes in this set.

#### Change Set B — DB Helper (no dependencies)

Add to `backend/src/db/database.ts`:

```typescript
export function getMaxSequenceNumber(sessionId: string): number {
  const row = getDb()
    .prepare('SELECT COALESCE(MAX(sequence_number), 0) AS maxSeq FROM session_output WHERE session_id = ?')
    .get(sessionId) as { maxSeq: number };
  return row.maxSeq;
}
```

Uses existing `idx_output_seq` index. No schema changes.

#### Change Set C — Stable Claude IDs (depends on A)

In `claude-code-jsonl-parser.ts`, replace `randomUUID()` with deterministic IDs:

- `parseUserEntry`: assign ID `${entry.uuid}-u-${blockIndex}` per block (0-based)
- `parseAssistantEntry`: assign ID `${entry.uuid}-a-${blockIndex}` per block (0-based)
- String content (no blocks): ID is `${entry.uuid}-u-0`

Remove the `randomUUID` import if no longer used elsewhere in the file.

**Edge case**: If `entry.uuid` is missing (malformed line), fall back to `randomUUID()` and log a warning with sessionId + line preview.

#### Change Set D — Copilot Line ID Parameter (no dependencies)

In `events-parser.ts`, update `parseJsonlLine` signature:

```typescript
export function parseJsonlLine(
  line: string,
  sessionId: string,
  sequenceNumber: number,
  lineId?: string
): SessionOutput | null
```

Inside: use `lineId ?? randomUUID()` for the output `id`. No other changes.

#### Change Set E — Claude Watcher (depends on A, B, C)

In `claude-jsonl-watcher.ts`:

1. Import `TAIL_BYTES` from `watcher-constants.ts`
2. Import `getMaxSequenceNumber` from `database.ts`
3. In `watchFile`: change initial file position from `0` to `Math.max(0, fileSize - TAIL_BYTES)`:
   ```typescript
   const { size: fileSize } = await fsStat(jsonlPath);
   this.filePositions.set(sessionId, Math.max(0, fileSize - TAIL_BYTES));
   ```
4. In `watchFile`: initialize sequence counter from DB:
   ```typescript
   this.sequenceCounters.set(sessionId, getMaxSequenceNumber(sessionId) + 1);
   ```
5. Add structured log line on attach: `[ClaudeWatcher] attach sessionId=... fileSize=... startPos=... startSeq=...`

#### Change Set F — Copilot Watcher (depends on A, B, D)

In `copilot-cli-detector.ts`:

1. Remove imports: `statSync`, `openSync`, `readSync`, `closeSync`
2. Add imports: `stat as fsStat`, `open as fsOpen` from `fs/promises`
3. Import `TAIL_BYTES` from `watcher-constants.ts`
4. Import `getMaxSequenceNumber` from `database.ts`
5. Remove `deleteSessionOutput` call from `watchEventsFile`
6. Initialize sequence counter from DB in `watchEventsFile`:
   ```typescript
   this.sequenceCounters.set(sessionId, getMaxSequenceNumber(sessionId) + 1);
   ```
7. Make `readNewLines` async; convert all sync I/O to async equivalents:
   ```typescript
   private async readNewLines(sessionId: string, filePath: string): Promise<void> {
     const { size: currentSize } = await fsStat(filePath);
     ...
     const fh = await fsOpen(filePath, 'r');
     const buffer = Buffer.alloc(currentSize - lastPos);
     await fh.read(buffer, 0, buffer.length, lastPos);
     await fh.close();
     ...
   }
   ```
8. Track byte offset of each line within the buffer; pass `${sessionId}-${lastPos + lineStartOffset}` as `lineId` to `parseJsonlLine`:

   ```typescript
   let byteOffset = 0;
   const lines: Array<{ text: string; offset: number }> = [];
   for (const part of newContent.split('\n')) {
     if (part.trim()) lines.push({ text: part, offset: lastPos + byteOffset });
     byteOffset += Buffer.byteLength(part, 'utf-8') + 1; // +1 for '\n'
   }
   ```

9. Make `watchEventsFile` async; await the initial `readNewLines` call:
   ```typescript
   private async watchEventsFile(sessionId: string, dirPath: string): Promise<void> {
     ...
     await this.readNewLines(sessionId, eventsFile);
     ...
   }
   ```
10. Update the chokidar onChange handler to handle the promise:
    ```typescript
    watcher.on('change', () => { this.readNewLines(sessionId, eventsFile).catch(() => {}); });
    ```
11. Update `processSessionDir` (or its caller) to await `watchEventsFile`.
12. Add structured log line: `[CopilotDetector] attach sessionId=... fileSize=... startPos=... startSeq=...`

### Function Size Guard

After the async conversion, `readNewLines` in `copilot-cli-detector.ts` will grow. Extract line-byte-offset tracking into a private helper:

```typescript
private splitLinesWithOffsets(
  buffer: Buffer,
  baseOffset: number
): Array<{ text: string; offset: number }>
```

This keeps `readNewLines` under 50 lines (§III).
